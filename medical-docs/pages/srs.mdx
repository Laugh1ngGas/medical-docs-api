# Software Requirement Specification

## 1. Introduction

**1.1 Purpose:**
This document defines the functional and non-functional requirements for a web platform for intelligent meal planning using personalization algorithms. The purpose of the system is to automate the process of generating personalized menus for users based on their available ingredients, culinary preferences, and dietary restrictions.

1.2 Scope:
The system is being developed to address the challenge of daily meal planning in the context of a fast-paced lifestyle and inflation. The platform will enable users to:
- Reduce the time spent searching for recipes and making dietary decisions.
- Optimize food expenses.
- Minimize food waste by using ingredients already available at home.
  In the future, the platform's architecture will support monetization and integration with grocery delivery services.

## 2. Overall Description

2.1 Operating Environment:
The software product is based on a modern client-server architecture:
- Backend: The server-side components and REST API are developed using C#.
- Database: The Supabase cloud platform (based on PostgreSQL) is used to store relational data (users, recipes, ingredients) and manage authentication.
- Frontend: A responsive web interface accessible via modern browsers (Chrome, Safari, Edge) on desktop and mobile devices.

2.2 User Characteristics:
- Unregistered user: Has access to the general catalog of popular recipes and the platform’s information page.
- Registered user: Has a personal account, can specify available ingredients, set dietary restrictions, receive personalized recommendations, and generate shopping lists.
- System administrator: Has access to manage the recipe database, ingredients, and user accounts.

## 3. System Features

3.1. User Profile Management
- Registration, authentication, and password recovery (powered by Supabase Auth).
- Storage of personal data: preferences regarding world cuisines, allergies, dietary restrictions (e.g., vegan, gluten-free), and target calorie intake.

3.2. Virtual Fridge (Inventory Management)
- Add, edit, and delete ingredients that the user currently has at home.
- Option to specify the quantity and expiration date of items.

3.3. Smart Menu Generation
- Operation of a recommendation algorithm that selects recipes from the database by matching available ingredients (from section 3.2) and user restrictions (from section 3.1).
- Sorting results by relevance (percentage of ingredient matches).

3.4. Shopping List Module (Shopping List Generator)
- Automatically generates a shopping list based on a selected recipe or a multi-day menu.
- The system should calculate the difference between the ingredients needed for the recipe and those already in the "virtual fridge."

3.5. Recipe Catalog
- View detailed information about a dish: photos, preparation steps, nutritional information (calories, protein, fat, carbohydrates), and cooking time.
- Save recipes to the "Favorites" category.

## 4. Non-Functional Requirements

4.1. Performance:
- The time required to generate a personalized menu must not exceed 3 seconds.
- The server-side must be able to handle concurrent requests through asynchronous processing in C#.

4.2. Security:
- All API requests involving personal data must be secured using a Bearer Token (JWT) generated by Supabase.
- User passwords are not stored in plain text (hashed at the Supabase level).

4.3. Reliability & Scalability:
- The database architecture must support the addition of new external APIs (e.g., to retrieve current prices at supermarkets) without altering the existing structure.
