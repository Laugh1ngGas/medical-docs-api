# Software Requirement Specification

## 1. Introduction

**1.1 Purpose:**
This document defines the functional and non-functional requirements for a web platform for cryptocurrency price tracking with an integrated crypto wallet. The purpose of the system is to provide users with a convenient tool for monitoring cryptocurrency markets, managing their digital assets, and analyzing portfolio performance in real time.

**1.2 Scope:**
The system is being developed to address the growing need for accessible and user-friendly cryptocurrency management tools. The platform will enable users to:

- Track real-time cryptocurrency prices and market trends.
- Manage a personal crypto portfolio.
- Analyze asset performance and historical data.
- Simulate or manage transactions (buy/sell operations).
- Make informed investment decisions.

In the future, the platform's architecture will support monetization and integration with external services such as exchanges and blockchain wallets.

## 2. Overall Description

**2.1 Operating Environment:**
The software product is based on a modern client-server architecture:

- Backend: Server-side components and REST API developed using Node.js / Express.
- Database: Cloud database (e.g., PostgreSQL, Supabase, or Firebase) used to store users, transactions, and portfolio data.
- Frontend: A responsive web interface built with React.js, accessible via modern browsers (Chrome, Firefox, Edge, Safari).

**2.2 User Characteristics:**

- Unregistered user: Can view general cryptocurrency market data and basic analytics.
- Registered user: Has a personal account, can manage a crypto portfolio, track assets, and perform transactions.
- System administrator: Has access to system configuration, monitoring tools, and user management.

## 3. System Features

**3.1 User Account Management**

- Registration, authentication, and password recovery.
- Secure login/logout functionality.
- Storage of user profile data.

**3.2 Cryptocurrency Market Tracking**

- Display a list of cryptocurrencies with key metrics:
  - Current price
  - 24h price change
  - Market capitalization
  - Trading volume

- Search and filtering functionality.

**3.3 Price Charts & Analytics**

- Display historical price charts.
- Ability to select time period (day, week, month).
- Visualization of trends.

**3.4 Crypto Wallet (Portfolio Management)**

- Add and manage crypto assets.
- View total portfolio balance.
- Automatic calculation of asset values.

**3.5 Transaction Management**

- Add transactions (buy/sell).
- Store transaction details (amount, price, date).
- View transaction history.

**3.6 Watchlist Feature**

- Add cryptocurrencies to favorites.
- Quick access to selected assets.

## 4. Non-Functional Requirements

**4.1 Performance:**

- Response time should not exceed 2 seconds.
- The system should support at least 1000 concurrent users.

**4.2 Security:**

- All requests must use HTTPS.
- Authentication via JWT tokens.
- Passwords must be securely hashed.
- Protection against common vulnerabilities (SQL Injection, XSS, CSRF).

**4.3 Reliability & Scalability:**

- System uptime should be at least 99%.
- The architecture must support scaling and future expansion.

## 5. External Interface Requirements

**5.1 User Interface:**

- Responsive and user-friendly design.
- Dashboard with key metrics.

**5.2 API Interface:**

- Integration with external cryptocurrency APIs (e.g., CoinGecko).

**5.3 Database Interface:**

- Storage of users, wallets, transactions, and watchlists.
